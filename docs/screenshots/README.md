# Screenshots для README

`README.md` и `README.ru.md` ссылаются на один файл:

- **`main-window.png`** — главное окно приложения, **рекомендуемая
  ширина 1200-1400 px**, тёмная тема, подключённое состояние
  (большая янтарная кнопка «Отключить», виден активный конфиг,
  статус «Подключено · …»). На гитхабе картинка ужмётся, но
  большой исходник даёт чёткость на ретине.

## Анонимизация (важно — публикуется под GPL)

Перед коммитом проверь что на скриншоте не видно:

- 🚫 **Реального IP твоего VPN-сервера** — для скриншота используй
  тестовый конфиг с обобщённым адресом (например `228.228.228.228:BIBA`).
- 🚫 **Subscription-URL с твоим private token'ом** (если он отображён
  где-то в Settings).
- 🚫 **`C:\Users\<твоё-имя>\...`** в path-полях.

Скриншот публикуется на github.com под GPL вместе с кодом — то, что
заблюришь, никто не увидит; то, что не заблюришь — проиндексирует
Google.

## Опционально потом

Если когда-нибудь захочется добавить второй кадр в README — хороший
кандидат `tray.png` (раскрытое меню системного трея с топ-3
quick-connect). Снимать: правый-клик по иконке KaproTUN в трее → пока
меню открыто, `PrtSc` → вставь в Paint → обрежь до меню → положи как
`docs/screenshots/tray.png`. Под него блок в README сейчас не
зарезервирован — добавлять надо вручную обоим файлам.

## Если файла нет

Hero-картинка вверху README показывается как broken-image иконка.
README сам не ломается, но GitHub-страница выглядит как «work in
progress». Чем быстрее положишь `main-window.png` — тем раньше
проект перестанет выглядеть полу-готовым.
